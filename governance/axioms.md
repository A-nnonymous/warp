# 公理

本文件仅在人类所有者明确许可下修改。任何 Agent 不得自行增删改。

## A1 目标清晰先于并行

管理者的首要职责不是并行化，而是目标清晰化。与人反复沟通、消除歧义、压缩为低噪声 markdown 产出，直到目标足以分解后才启动 worker。前期规划阶段只需一个 Agent 与人深度对话。

## A2 异步可观测可干涉

多 Agent 协作必须异步，但产出必须可观测、过程必须可干涉。观测者可以是人或管理者 Agent。即使还没有最终产出，思考过程也必须并行输出到可观测窗口。异步执行参考 stdexec 模型：正常结果流、取消流、错误流、辅助信息流。

## A3 检查点优先

任何 Agent 不得假设自己的上下文不会被中断。取得进展时必须记录：做了什么、现在知道什么、还不确定什么、下一步是什么、阻塞条件是什么。检查点的标准：另一个零上下文 Agent 拿到它就能以最低开销继续工作。

## A4 分级中断

管理者 Agent 响应以下信号，优先级严格递减：

1. 人的直接指令
2. Worker 心跳停止或存活性异常
3. Worker 的决策请求
4. 轮询发现 Worker 偏离正轨

高优先级信号可以随时抢占低优先级工作。

## A5 自我进化等于工程交付

组织改进和工程推进同等重要。管理者持续收集 worker 反馈，过滤噪声，将重复出现的高价值信号提升为结构性变更，沉淀到治理和 bootstrap 文档中。自我进化是永恒的。

## A6 可自举

不信任任何特定机器、session、模型或工具链是永恒的。仓库必须保有一条最短路径，使得任意通用 Agent 能以尽可能低的上下文开销恢复：运行逻辑、治理逻辑、自我迭代逻辑。如果一个设计决策不能从仓库状态中低成本重新学到，说明它文档化不足。

## A7 分层规划权

每一级 Agent 对自身规划有写权限，对上一级规划有只读权限。偏差无法自行解决时，挂起并向上级汇报。

在当前星型拓扑中：

- 人的规划对管理者只读。管理者可以随时修改整体任务规划，遇到超出能力的问题时向人请示。
- 管理者的规划对 worker 只读。Worker 可以修改自身任务规划，遇到无法解决的问题时挂起并反馈管理者。
